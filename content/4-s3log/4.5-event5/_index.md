# Cloud Mastery 2026 - #3

**Time:** April 11, 2026  
**Location:** Lot E2a-7, D1 Street, Hi-Tech Park, Long Thanh My Ward, Thu Duc City, HCMC

---

### About the Event

![Networking Session](/AWS_Intern-Report/images/z7713504050761_54bb9ca5954d69975fef6907cce2be73.jpg)

The "Cloud Mastery 2026 - #3" event brought in-depth knowledge about building secure, resilient, and well-architected network infrastructures on the cloud. Across three sessions, experts guided attendees in detail on how to design Virtual Private Clouds (VPC), manage Identity and Access Management (IAM) best practices, and deploy comprehensive network and application protection strategies against modern cyber threats.

#### 09:00 AM - 10:00 AM | Networking on AWS
**Speakers:** Lam An Thinh & Nguyen Phan Quoc Viet

The first session addressed the core fundamentals of cloud networking, illustrating how to structure and secure traffic flow within an AWS environment[cite: 14].

**Key Takeaways:**
* **AWS Networking Fundamentals:** Explored the foundational concepts of VPC, CIDR, Subnets, and the importance of controlling the "Blast Radius"[cite: 15, 16, 29].
* **Traffic Routing:** Detailed the roles of Internet Gateways (IGW), Route Tables, and Elastic IPs in managing ingress and egress traffic[cite: 17, 40, 41].
* **NAT Gateway:** Highlighted the differences between Zonal and Regional NAT Gateways, explaining how SNAT and PAT mechanisms provide secure, outbound-only internet access for private subnets[cite: 66, 70, 71, 79].
* **Security Groups vs. NACL:** Distinguished between Security Groups (stateful, ENI-level, allow-only rules) and Network ACLs (stateless, subnet-level, evaluated in numerical order with explicit deny support)[cite: 112, 116, 120, 126, 128, 142].

**Speakers:** Huynh Hoang Long & Dang Thi Minh Thu 

The second session focused on solving the challenges of access control, emphasizing the zero-trust approach and centralized permission management in AWS.

**Key Takeaways:**
* **IAM Best Practices:** Emphasized the least privilege principle, enabling MFA for all users, deleting root access keys, and rotating passwords regularly[cite: 351, 352, 359, 360].
* **Single Sign-On (SSO):** Discussed utilizing SSO for multi-account integration, providing one login for multiple apps and centralized access management[cite: 362, 363, 364].
* **SCPs vs. Permission Boundaries:** Explained that Service Control Policies (SCPs) act as organizational policies to filter maximum permissions [cite: 370, 372][cite_start], whereas Permission Boundaries apply specific limits to Users/Roles within an individual account[cite: 377].
* **Credential Spectrum:** Recommended avoiding long-term IAM User Access Keys in favor of short-term credentials generated by the AWS Security Token Service (STS)[cite: 380, 381, 383].

#### 11:00 AM - 12:00 PM | AWS Network & Application Protection
**Speaker:** Lam Tuan Kiet

The final session brought a new perspective on shielding workloads from vulnerabilities and malicious traffic using native AWS security services.

![Security Session](/AWS_Intern-Report/images/z7713813165479_b55020e7b9b9abd8eed13bffdb7ffcae.jpg)
![IAM Session](/AWS_Intern-Report/images/z7713813161897_9bfbe2dcab727c13a2462533faf0481b.jpg)

![IAM Session](/AWS_Intern-Report/images/z7713819274397_8de82aab29f4657784e03aaefbb4b04e.jpg)

**Key Takeaways:**
* **AWS WAF (Web Application Firewall):** Protects web applications from common web exploits by filtering HTTP/HTTPS requests, specifically preventing attacks like SQL Injection and Cross-Site Scripting (XSS)[cite: 405, 406, 408, 409].
* **AWS Shield:** Provides managed protection against DDoS attacks to ensure high availability, offering both a free Standard tier and an Advanced tier for enhanced protection[cite: 417, 419, 420].
* **AWS Network Firewall:** Deploys security boundaries within VPCs, offering both stateful and stateless traffic filtering alongside intrusion prevention[cite: 428, 430, 431].
* **AWS Firewall Manager:** Serves as a centralized security management tool across AWS accounts, automating rule deployments for WAF, Shield, and Network Firewall to ensure consistent multi-account policies[cite: 439, 440, 444].
